What Makes This Different

This isn't a CRM that happens to have AI bolted on. AI is the default layer. Every outbound message is written by Claude using the borrower's full profile. Every page shows AI-generated intelligence. When AI is unavailable, static templates kick in automatically — the system never stops working.

AI Intelligence On Every Page

Claude is embedded in every screen of the CRM. Here's what each page does:

☀

Dashboard — Morning Brief AI Sonnet

One click generates your daily pipeline intelligence: today's priorities (specific deals by name), closing watch, stale alerts with recommended action type, and a win-of-the-week highlight. Uses real data from all 200+ active deals.

👤

Contact Detail — AI Summary + Message Writer AI Haiku

Summary: Opens with relationship health (GREEN/YELLOW/RED), a 2-sentence snapshot of who they are, the #1 next action, talking points for your next call, and risk flags.
Message Writer: Pick a purpose (welcome, follow-up, DPA info, rate update, etc.) and Claude writes a personalized SMS or email using their full profile — credit score, loan type, location, stage, everything.

📊

Deal Detail — AI Deal Review AI Sonnet

One-click health check on any deal: GREEN/YELLOW/RED status, specific blockers, 3 concrete next steps with timeline, risk factors, estimated days to close, and confidence level. Works on every deal, not just stale ones.

📈

Reports — Pipeline Analysis AI Sonnet

AI reads your numbers and writes the headline: trend direction (UP/FLAT/DOWN), 3-4 data-driven insights, where deals are getting stuck and why, top recommendation for the week, and revenue forecast.

💬

Inbox — Smart Reply Drafts Ready

When a message comes in, Claude drafts 3 reply options in different tones (professional, friendly, brief). Pick one, edit if needed, send. Component is built and ready — wires in when iMessage/email connects.

⚙

Automations — AI-Personalized Messages AI Haiku

Every outbound borrower message is rewritten by Claude at queue time. A first-time buyer in Homestead gets a completely different welcome than a refi in Boca. If AI is unavailable, the static template sends instead — no message ever gets stuck.

AI Safety Net

AI never blocks the CRM

If Claude goes down (rate limits, outages, etc.), the system keeps working:

  • Circuit breaker: After 3 failures, AI skips for 5 minutes then auto-recovers
  • Tight timeouts: Haiku 30s, Sonnet 60s (not 3 minutes)
  • Static fallback: 41 pre-written templates always work as backup
  • Non-blocking: Automations fire instantly; AI personalization is best-effort
  • Health endpoint: /api/ai/status shows breaker state at any time

Tech Stack

Frontend

Next.js 16 + React 19 + TypeScript + Tailwind v4. dnd-kit for Kanban, recharts for charts, lucide-react icons.

Backend CRM

Twenty CRM (open-source, self-hosted Docker). 4,625 contacts, 283 deals, 23 stages, 78-tool MCP server.

AI Engine

Claude CLI via Pro Max OAuth ($0 per call). Haiku for speed, Sonnet for depth. No API tokens needed.

Hosting

Mac Mini via Cloudflare Tunnel at crm.homerfg.com (port 3004). Twenty Docker at localhost:3000.
